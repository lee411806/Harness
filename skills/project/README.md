# Skill: Project

## Purpose
Capture project-specific context that isn't derivable from the code alone.

## Sections to Fill In

### Project Overview
> One paragraph: what this project does, who uses it, and why it exists.

### Tech Stack
> List languages, frameworks, databases, and key services in use.

### Key Contacts / Owners
> Who owns what area. Helps route questions to the right person.

### Active Initiatives
> Current goals or work streams. Update as priorities shift.

### Known Constraints
> Decisions already made that are not up for debate (compliance requirements, vendor lock-in, legacy systems, etc.)

### Glossary
> Domain terms that have specific meanings in this project.
